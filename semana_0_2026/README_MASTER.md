## 🎯 VISÃO GERAL DA SEMANA 0

### Propósito Pedagógico

A **Semana 0** é a porta de entrada dos alunos na Innova Academy, desenhada para:

✅ **Nivelar conhecimentos prévios** (VARK, diagnóstico técnico)  
✅ **Estabelecer fundamentos éticos** desde o primeiro dia  
✅ **Gerar artefato tangível** em 2 horas (senso de realização imediata)  
✅ **Despertar curiosidade** para as 16 lições seguintes  
✅ **Conectar com famílias** através de homework e family work

## 🎨 DECISÕES DE DESIGN IMPLEMENTADAS

### 1. Curiosity (6+ anos)
**Sugestões Implementadas:**
- ✅ VARK simplificado (15 min em vez de 30)
- ✅ Ética com apenas 2 cenários visuais
- ✅ Projeto expandido para 70 min (garantir tempo suficiente)
- ✅ Plano B robusto (galeria pré-gerada + desenho híbrido + vídeo)
- ✅ Ferramenta: Bing Image Creator (mais simples que Midjourney)

**Diferenciais:**
- Cartões de planejamento de avatar (evitar tentativas aleatórias)
- Banco de Palavras Mágicas (vocabulário ilustrado)
- Ritual de Gratidão Tecnológica Verde

---

### 2. Discovery (9+ anos)
**Sugestões Implementadas:**
- ✅ **Opção B**: Classificador de Objetos (mais rápido que emoções)
- ✅ Trabalho em DUPLAS (acelera coleta de dados)
- ✅ Apenas 3 classes (não 4) para caber no tempo
- ✅ Objetos do cotidiano escolar (caneta, caderno, garrafa)
- ✅ Plano B: dataset pré-coletado + modo offline

**Diferenciais:**
- Jogo "Enganando a Máquina" (viés de dados prático)
- Checklist de Treinamento estruturado
- Desafios progressivos (Bronze/Prata/Ouro)
- Repositório coletivo de modelos (Google Sheets)

---

### 3. Pioneer (12+ anos)
**Sugestões Implementadas:**
- ✅ Chatbot simplificado (lógica condicional, não API externa)
- ✅ 3 níveis de templates (iniciante 80% pronto, avançado 20%)
- ✅ IA como assistente (ChatGPT/Gemini), não fazer o trabalho sozinha
- ✅ Demonstração progressiva (V1 → V2 → V3)
- ✅ Plano B: Thonny offline + programação em papel

**Diferenciais:**
- Estudo de caso "O Bug Inocente" (ética prática)
- "Compromisso do Desenvolvedor" assinado
- Checklist de Debugging visual
- Pair programming para validação

---

### 4. Challenger (14+ anos)
**Sugestões Implementadas:**
- ✅ **Opção A**: Dashboard de Exploração (dataset pré-limpo)
- ✅ Foco em visualização, não análise de sentimento complexa
- ✅ Notebook template 70% pronto (células guiadas)
- ✅ Dataset social relevante (200-500 linhas)
- ✅ Plano B: Excel/Sheets + análise manual

**Diferenciais:**
- Debate "Algoritmo de Empregos Justo" (caso Amazon real)
- Checklist de Impacto Ético personalizado
- Banco de 30 perguntas analíticas
- Apresentação TED de 1 minuto

## 📈 CRITÉRIOS DE SUCESSO (KPIs)

### Curiosity
- ≥90% criaram e salvaram avatar
- ≥70% explicam o que é "prompt"
- 100% formularam Regra de Ouro

### Discovery
- ≥85% criaram modelo funcional
- ≥70% coletaram ≥50 amostras/classe
- ≥90% identificaram viés no experimento falho

### Pioneer
- ≥85% executam chatbot sem erros críticos
- ≥75% implementaram if/elif/else corretamente
- ≥60% resolvem erros simples autonomamente

### Challenger
- ≥85% criaram análise completa
- ≥80% criaram 3+ gráficos rotulados
- ≥75% extraíram 3+ insights relevantes

## 📝 HOMEWORK E FAMILY WORK

Todos os cursos incluem **2 tipos de tarefas pós-aula**:

### 📚 Homework (Individual)
Expandir o artefato criado na aula:
- Curiosity: Encontrar 3 tecnologias com IA em casa
- Discovery: Fotografar 3 exemplos de classificação automática
- Pioneer: Adicionar funcionalidades ao chatbot (3 níveis)
- Challenger: Expandir dashboard com novas visualizações (3 níveis)

### 👨‍👩‍👧 Family Work (Com a Família)
Conectar aprendizado com contexto familiar:
- Curiosity: Explicar avatar e coletar ideias da família
- Discovery: Ensinar familiar a treinar IA
- Pioneer: Demonstrar chatbot e implementar sugestão
- Challenger: Apresentar dashboard e coletar ideias de análise

## 🚨 PLANOS B (CONTINGÊNCIAS)

Todos os cursos têm **múltiplos planos B** para cenários de falha:

### Cenários Cobertos
1. ❌ Internet instável/fora
2. ❌ Ferramenta principal não funciona
3. ❌ Webcam/dispositivos com problemas
4. ❌ Aluno muito travado (não avança)

### Estratégias de Mitigação
- **Offline:** Versões locais de ferramentas (Thonny, datasets salvos)
- **Analógico:** Atividades em papel (pseudocódigo, análise manual)
- **Demonstração:** Vídeos pré-gravados + discussão conceitual
- **Simplificação:** Templates 90% prontos para casos extremos

## 🎓 VOCABULÁRIO-CHAVE POR CURSO

### Curiosity (6+)
- **Inteligência Artificial (IA):** Máquina que aprende
- **Prompt:** Palavras mágicas para falar com IA
- **Avatar:** Personagem digital
- **Geração:** Quando a IA cria algo novo

### Discovery (9+)
- **Machine Learning:** IA que aprende com exemplos
- **Classificação:** Organizar em categorias
- **Dataset:** Conjunto de dados
- **Treinamento:** Processo de aprendizado da IA
- **Viés:** Erro por dados limitados

### Pioneer (12+)
- **Função:** Bloco de código reutilizável
- **Condicional:** Estrutura de decisão (if/else)
- **Loop:** Repetição de código
- **Debugging:** Encontrar e corrigir erros
- **Parâmetro:** Informação passada para função

### Challenger (14+)
- **Dataset:** Conjunto organizado de dados
- **Análise Exploratória (EDA):** Primeira fase de análise
- **Visualização:** Representação gráfica
- **Insight:** Descoberta extraída dos dados
- **Correlação:** Relação entre variáveis
- **Viés Algorítmico:** Discriminação sistemática
